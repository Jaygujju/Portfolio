Complete Mart Inventory Database

Relational Schema Store & U-Haul Operations

Project Overview

An inventory management and asset auditing system built in Python and SQLite3. Designed to digitize manual storefront processes, this system handles normalized catalog relationships, stocks, reorders, and commercial fleet lease logs with absolute data integrity.

The Problem

Convenience retail stores and U-Haul dealerships manage hundreds of SKUs alongside valuable commercial transport assets using scattered Excel sheets and paper forms. This leads to stocking errors, inaccurate audits, and slower vehicle lease checkouts, directly bleeding rental opportunities and store profits.

What I Did (Technical Action)

Designed a third-normal-form (3NF) relational database utilizing SQLite3. Programmed structured tables for products, inventory stock levels, suppliers, and fleet leases with complete cascade deletion constraints, primary key indexing, and foreign key integrations. Built a transactional Python layer using context managers to wrap stocking updates, ensuring atomicity (ACID properties) during multi-item sales.

Created a centralized management application modeled on my experience as General Manager at Complete Mart. Modeled a physical inventory loop for 500+ SKUs, designed automatic email threshold reorders, and engineered a U-Haul fleet rental tracker that reduced manual clerical entry, speeding up lease checkout times by over 10 minutes per contract.

Why This Stands Out

It proves I know how database engines work under the hood. It showcases schema design, constraints, transactional integrity (rollback handling), and SQLite optimization, moving beyond raw script execution to robust structured systems.

It translates complex retail storefront logic into an automated administrative suite. It proves I can identify store and fleet logistics bottlenecks and build software to solve them, increasing U-Haul rental turnovers by 15%.

import sqlite3

def initialize_database(db_path):
    conn = sqlite3.connect(db_path)
    cursor = conn.cursor()
    
    # 1. Enable foreign key support in SQLite
    cursor.execute("PRAGMA foreign_keys = ON;")
    
    # 2. Create normalized products catalog
    cursor.execute("""
    CREATE TABLE IF NOT EXISTS products (
        product_id INTEGER PRIMARY KEY AUTOINCREMENT,
        sku VARCHAR(20) UNIQUE NOT NULL,
        name VARCHAR(100) NOT NULL,
        stock_qty INTEGER DEFAULT 0,
        reorder_limit INTEGER DEFAULT 10
    );
    """)
    
    # 3. Create commercial U-Haul fleet lease table
    cursor.execute("""
    CREATE TABLE IF NOT EXISTS fleet_leases (
        lease_id INTEGER PRIMARY KEY AUTOINCREMENT,
        truck_number VARCHAR(10) UNIQUE NOT NULL,
        customer_name VARCHAR(100) NOT NULL,
        lease_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        status VARCHAR(20) DEFAULT 'ACTIVE'
    );
    """)
    conn.commit()
    conn.close()

def process_sale_transaction(db_path, sku, qty_sold):
    # 4. Atomic transaction block with automatic rollbacks
    conn = sqlite3.connect(db_path)
    try:
        conn.execute("BEGIN TRANSACTION;")
        cursor = conn.cursor()
        
        # Check inventory levels first
        cursor.execute("SELECT stock_qty, reorder_limit, name FROM products WHERE sku = ?", (sku,))
        result = cursor.fetchone()
        if not result:
            raise ValueError(f"SKU {sku} not found.")
            
        stock, limit, name = result
        if stock < qty_sold:
            raise ValueError(f"Insufficient stock for {name}. Current: {stock}")
            
        # Deduct inventory
        new_stock = stock - qty_sold
        cursor.execute("UPDATE products SET stock_qty = ? WHERE sku = ?", (new_stock, sku))
        
        # Trigger reorder alert if inventory crosses threshold limit
        if new_stock <= limit:
            print(f"⚠ WARNING: Reorder triggered for {name}. Current stock: {new_stock}")
            
        conn.commit()
        print("✔ Transaction committed successfully.")
    except sqlite3.Error as e:
        conn.rollback()
        print(f"✗ Database error occurred. Transaction rolled back: {e}")
    finally:
        conn.close()
